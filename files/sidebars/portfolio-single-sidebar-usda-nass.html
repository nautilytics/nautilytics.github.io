USDA/NASS Dynamic Mapping Tool

Context
USDA/NASS wanted to update their static mapping visualizations, so they contracted with me to build a single-page, dynamic, interactive mapping application on top of their QuickStats agricultural data API.
Data
USDA/NASS QuickStats API with Census and Survey agricultural data at the national, state, and county levels as far back as the late 1800s.
Tech Notes
  • The flexible protocol buffer and GeoJSON tile server for this project gives users the ability to create custom choropleth map tiles on top of any mapping platform.
  • The front-end is built using d3.js, jQuery, Handlebars, Mapbox, and Leaflet.
  • The back-end technologies include PostgreSQL, PostGIS, node.js, Express, Mapnik, Redis, Fastly, and Nginx.
